Models:
In Blazor you can set up model restrictions with square brackets.

[Required]
public string Name { get; set; }

[Range(upper, lower)]
public int Age { get; set; }

You can make a field nullable with a question mark by the data type
public int? Age { get; set; }

to initialize a field with a default, add an equals after the setter
public string Name { get; set; } = ";

To put a model into a form in a .razor doc?
we have this <EditForm> tag
<EditForm Model="model"> this attribute Model allows you to specify a variable to be set as the model of this form

Blazor has baked in input field components
<InputText />
We need to bind values to input fields.
<InputText @bind-Value="model.Name"/>

We can pich a method to launch on submission with
<EditForm Model="model" OnValidSubmit="">

There are premade components that enforce and display validation
<DataAnnotationsValidator /> Enforces model validation

<ValidationSummary/> Shows a little validation message for the whole form

<ValidationMessage For="@(() => model.Name)"/> shows a validation message for a specified field
